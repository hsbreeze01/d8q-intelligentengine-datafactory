## Context

当前系统有两条信息路径：
1. **任务管理页**（⚙️ 任务）：`loadTasks()` 渲染采集任务表格，有 track badge 和"管理关键词"按钮
2. **赛道页**（🔥 赛道）：`loadTrack()` 渲染赛道卡片，显示热度、资讯数等统计

问题：
- 任务表格缺少类型区分，赛道/自定义任务视觉上一致
- "管理关键词"按钮在任务行上，但关键词是赛道属性。同一赛道多个任务 → 重复按钮
- 赛道页是赛道信息的自然入口，但没有任何管理功能

## Goals / Non-Goals

**Goals:**
- 任务列表可直观区分赛道采集 vs 自定义采集
- 关键词管理入口迁移到赛道页，语义清晰：赛道 → 关键词
- 任务行只做任务操作（执行/暂停/删除），不做赛道属性管理

**Non-Goals:**
- 不改变赛道页的整体布局和热度图表
- 不修改后端 API
- 不涉及赛道本身的增删改

## Decisions

### 1. 任务类型列：新增"类型"表头列

**选择**：在 `<th>` 增加"类型"列，用彩色 badge 显示 `赛道采集`（蓝色）/ `自定义采集`（灰色）。

**理由**：与现有 badge 风格一致（创作/发布 badge 在创作任务表中已有先例），一眼可识别。

**实现**：根据 `t.track_id` 是否存在判断类型。有 track_id → 从 trackMap 取赛道名显示"赛道采集"；无 → "自定义采集"。

### 2. 移除任务行的"管理关键词"按钮

**选择**：删除 `loadTasks()` 中 badge 旁边的"管理关键词"按钮，只保留赛道 badge。

**理由**：关键词属于赛道，不属于任务。管理入口在赛道页更合理，避免同一赛道多任务的重复入口。

### 3. 赛道卡片增加"管理关键词"链接

**选择**：在赛道页 `loadTrack()` 的每个赛道卡片（`.tc`）底部增加一个"🔑 管理关键词"链接/按钮。

**理由**：
- 赛道卡已有资讯数/政策数等统计，关键词数量/管理是自然延伸
- 每个赛道只有一个卡片，一个入口，无重复
- 用户心智模型：赛道 → 关键词，更直观

**备选**：赛道卡片点击跳转到独立赛道详情页 → 过重，MVP 阶段不需要

## Risks / Trade-offs

- **赛道页点击管理关键词会离开赛道页** → 模态框 overlay，不导航离开，赛道页保持可见
- **任务行的赛道 badge 不再可交互** → 纯展示，如需跳转赛道页可后续迭代
