# Delta Spec: 策略组页面按线框图重构

## MODIFIED Requirements

### Requirement: 策略发现页按线框图重构

策略发现页 (`/strategy/discover/`) SHALL 按线框图 `page-discover` 布局重构，包含以下区域：

1. **顶部统计卡片**：4 格统计网格，展示可订阅策略组数、已订阅数、活跃事件数、平均命中率
2. **工具栏**：搜索输入框 + 类型筛选下拉（全部/技术面/量价/趋势）
3. **策略卡片网格**：`auto-fill minmax(320px, 1fr)` 自适应卡片布局
   - 每张卡片 SHALL 展示：名称、状态标签、描述、策略数量、命中率、今日信号数
   - 卡片 SHALL 显示关联策略标签（如 RSI 超卖、MACD 金叉等 tag 标签）
   - 已订阅卡片 SHALL 显示"已订阅"标签和"查看我的策略"按钮
   - 未订阅卡片 SHALL 显示"+ 订阅此策略"按钮
   - 订阅按钮点击后立即变为"✓ 已订阅"状态（disabled），无需刷新页面

#### Scenario: 用户浏览策略发现页

- **Given** 用户已登录
- **When** 用户访问 `/strategy/discover/`
- **Then** 页面展示 4 格统计卡片、搜索工具栏、策略卡片网格
- **And** 已订阅的策略卡片带有蓝色"已订阅"标签
- **And** 未订阅的策略卡片显示订阅按钮

#### Scenario: 用户搜索和筛选策略

- **Given** 策略发现页已加载
- **When** 用户在搜索框输入关键词或选择类型筛选
- **Then** 策略卡片网格实时过滤，仅显示匹配的策略组

#### Scenario: 用户订阅策略组

- **Given** 用户在策略发现页看到未订阅的策略组
- **When** 用户点击"+ 订阅此策略"按钮
- **Then** 按钮变为"✓ 已订阅"（disabled 状态）
- **And** 顶部统计卡片的"已订阅"数值 +1

---

### Requirement: 我的策略页按线框图重构

我的策略页 (`/strategy/my/`) SHALL 按线框图 `page-my` 布局重构：

1. **顶部统计卡片**：4 格统计 — 已订阅策略数、活跃事件数、关联板块涨幅（含涨跌箭头）、今日信号数
2. **按策略组分段**：每个已订阅策略组为一个独立 card 区块
   - 区块头部：策略组名称 + 策略数/命中率/状态 + 取消订阅按钮
   - 活跃事件列表：使用线框图 `event-card` 样式（图标 + 事件名 + 触发股票数 + 确认度星级 + 起始日期 + 关联资讯数 + 涨跌幅 + lifecycle badge）
   - 无活跃事件时显示空状态提示

#### Scenario: 用户查看我的策略

- **Given** 用户已登录且已订阅 2 个策略组
- **When** 用户访问 `/strategy/my/`
- **Then** 页面展示 4 格统计和按策略组分段的事件列表
- **And** 每个活跃事件卡片展示图标、名称、触发数、确认度、日期、涨跌幅、lifecycle badge
- **And** 点击事件卡片跳转到事件详情页

#### Scenario: 已订阅策略组无活跃事件

- **Given** 用户已订阅的策略组当前无活跃事件
- **When** 用户在我的策略页查看该策略组区块
- **Then** 区块内显示空状态（"暂无活跃事件"图标+文字）

### Requirement: 事件详情页按线框图三 Tab 重构

事件详情页 (`/strategy/events/<id>/`) SHALL 按线框图 `page-event` 重构为全屏页面布局：

1. **顶部工具栏**：返回按钮 + 事件名称 + lifecycle badge + 管理操作按钮（建议关闭/确认关闭）
2. **统计卡片**：4 格 — 触发股票数、确认度（星级可视化）、持续天数、板块累计涨跌幅
3. **三 Tab 布局**：
   - **🔬 微观数据**：股票快照卡片网格（`stock-snapshot-grid`），每张卡片展示股票名称+星级评分+指标键值对网格（KDJ K/D、RSI_6/12、量比、MACD DIF、涨跌幅、buy_star），底部"展开全部"按钮
   - **📊 宏观数据**：2 个 ECharts 图表（行业趋势指标聚合图 + 板块涨跌走势图）+ 每日跟踪统计表格（日期/触发股票/新增/消失/板块涨跌/均值评分/资讯关联）
   - **🔗 信息关联**：左右双栏布局 — 左侧 LLM 智能分析（事件类型+置信度+驱动因素+关联主题+摘要建议框）+ 右侧关联资讯流（来源+时间+相关性分数+标题+摘要）
4. **生命周期管理条**：底部展示 lifecycle 流转说明（跟踪中 → 建议关闭 → 已关闭）

#### Scenario: 用户查看事件微观 Tab

- **Given** 事件详情页已加载，存在 15 只触发股票
- **When** 用户查看"微观数据"Tab
- **Then** 展示股票快照卡片网格，每张卡片含股票代码名称、星级评分、8 个指标键值对
- **And** 默认显示前 5 张卡片，底部有"展开全部 N 只 →"按钮

#### Scenario: 用户查看事件宏观 Tab

- **Given** 事件详情页已加载
- **When** 用户切换到"宏观数据"Tab
- **Then** 展示 2 个 ECharts 图表（趋势指标聚合 + 板块涨跌走势）
- **And** 下方展示每日跟踪统计表格

#### Scenario: 用户查看事件信息关联 Tab

- **Given** 事件详情页已加载，LLM 分析已生成
- **When** 用户切换到"信息关联"Tab
- **Then** 左侧展示 LLM 智能分析结果（事件类型、置信度、驱动因素、关联主题、摘要建议）
- **And** 右侧展示按相关性排序的关联资讯流

### Requirement: 管理员策略列表页按线框图重构

管理员策略管理页 (`/strategy/admin/groups/`) SHALL 按线框图 `page-list` 重构：

1. **顶部统计卡片**：4 格 — 策略组总数、运行中数量、关联策略总数、今日平均收益
2. **工具栏**：搜索框 + 状态筛选下拉 + "新建策略组"按钮
3. **策略卡片网格**：替代现有的表格布局，每张卡片包含：
   - 名称 + 状态标签
   - 描述
   - 策略数/命中率/今日信号 meta
   - 关联策略 tag 列表
   - 操作区：编辑按钮 + 暂停/激活按钮 + 删除按钮

#### Scenario: 管理员查看策略列表

- **Given** 管理员已登录
- **When** 访问 `/strategy/admin/groups/`
- **Then** 页面展示统计卡片 + 搜索筛选工具栏 + 卡片网格
- **And** 每张卡片有编辑、暂停/激活、删除操作按钮

#### Scenario: 管理员从列表进入策略组详情

- **Given** 管理员在策略列表页
- **When** 管理员点击策略卡片（非按钮区域）
- **Then** 跳转到该策略组的详情页

---

### Requirement: 管理员策略组配置页补充信号规则区域

管理员策略组配置页 (`/strategy/admin/groups/<id>/edit`) SHALL 在现有表单基础上增加"信号规则"区块：

1. **买入阈值**：文本输入（如"≥ 3 个策略同时命中"）
2. **冷却期**：文本输入（如"同一股票 5 个交易日内不重复信号"）
3. 现有的策略关联表格 SHALL 展示每行策略的权重、阈值输入框

#### Scenario: 管理员编辑信号规则

- **Given** 管理员编辑已有策略组
- **When** 页面加载完成
- **Then** 信号规则区域显示已保存的买入阈值和冷却期配置
- **And** 关联策略表格每行包含可编辑的权重和阈值输入

## ADDED Requirements

### Requirement: 策略组详情页（新增）

新增管理员策略组详情页 (`/strategy/admin/groups/<id>/`)，按线框图 `page-detail` 布局：

1. **顶部工具栏**：返回按钮 + 策略组名称 + 状态标签 + 操作按钮（回测/运行管理/编辑）
2. **统计卡片**：4 格 — 策略数量、综合命中率、今日信号数（买入/卖出拆分）、累计收益
3. **活跃事件列表**：同线框图 `event-card` 样式，点击进入事件跟踪页
4. **双栏区域**：
   - 左栏：今日信号快照表格（股票/信号方向/触发策略/综合得分/时间）
   - 右栏：近 7 日信号趋势 ECharts 柱状图（买入/卖出堆叠）

#### Scenario: 管理员查看策略组详情

- **Given** 管理员已登录，存在策略组 ID=1
- **When** 管理员访问 `/strategy/admin/groups/1/`
- **Then** 页面展示统计卡片、活跃事件列表、信号快照表格、信号趋势图表

#### Scenario: 管理员从详情页导航到回测

- **Given** 管理员在策略组详情页
- **When** 点击"📊 回测"按钮
- **Then** 跳转到该策略组的回测页

---

### Requirement: 策略组回测页（新增）

新增管理员回测页 (`/strategy/admin/groups/<id>/backtest/`)，按线框图 `page-backtest` 布局：

1. **顶部工具栏**：返回按钮 + 标题 + 时间范围选择下拉 + "运行回测"按钮
2. **统计摘要**：6 格网格 — 累计收益、胜率、盈亏比、最大回撤、总交易次数、平均持仓天数
3. **双栏图表区域**：
   - 左栏：ECharts 收益曲线 vs 基准对比折线图
   - 右栏：ECharts 月度收益分布柱状图（正收益绿色，负收益红色）

#### Scenario: 管理员运行回测

- **Given** 管理员在回测页，选择"近 30 天"
- **When** 点击"运行回测"按钮
- **Then** 系统调用 Compass API 执行回测
- **And** 更新统计摘要和图表数据

---

### Requirement: 策略组运行管理页（新增）

新增管理员运行管理页 (`/strategy/admin/groups/<id>/run/`)，按线框图 `page-run` 布局：

1. **顶部工具栏**：返回按钮 + 标题 + 暂停/立即执行按钮
2. **统计卡片**：4 格 — 当前状态、下次执行时间、今日信号数、成功率（近7日）
3. **调度配置区**：执行频率下拉 + 执行时间输入 + 股票池描述
4. **运行历史时间线**：竖向时间线组件，每条记录显示执行时间 + 结果描述（成功/失败标记）

#### Scenario: 管理员查看运行历史

- **Given** 管理员访问运行管理页
- **When** 页面加载完成
- **Then** 运行历史以时间线形式展示，成功节点绿色、失败节点红色
- **And** 每条记录包含时间戳和执行结果描述

#### Scenario: 管理员立即执行策略扫描

- **Given** 策略组处于"运行中"状态
- **When** 管理员点击"🔄 立即执行"按钮
- **Then** 系统触发一次手动执行
- **And** 时间线顶部新增一条"执行中"记录

### Requirement: 策略组导航和路由扩展

1. `base.html` 侧边栏 SHALL 根据用户角色展示不同导航项：
   - 用户角色：策略发现、我的策略

2. `compass_pages.py` SHALL 新增以下路由：
   - `GET /strategy/admin/groups/<id>/` — 策略组详情
   - `GET /strategy/admin/groups/<id>/backtest/` — 回测页
   - `GET /strategy/admin/groups/<id>/run/` — 运行管理页
3. 新增对应 Jinja2 模板文件

#### Scenario: 管理员从策略列表导航到详情

- **Given** 管理员在策略列表页
- **When** 点击策略卡片
- **Then** 跳转到 `/strategy/admin/groups/<id>/` 详情页

#### Scenario: 面包屑正确显示当前位置

- **Given** 用户在任意策略子页面
- **When** 页面渲染完成
- **Then** 顶部面包屑正确显示导航路径（如"首页 / 策略组详情"）

## REMOVED Requirements

### Requirement: 移除管理员列表页的表格布局

管理员策略管理页 SHALL 不再使用纯表格布局展示策略组列表，改为线框图定义的卡片网格布局。现有表格行样式和相关的 CSS 类 SHALL 被替换。
