---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **信息密度优先** | 金融用户需要在一屏内获取最大信息量，减少翻页和跳转 |
| **数据驱动视觉** | 用图表和指标卡替代大段文字，关键数据一眼可见 |
| **操作路径最短** | 核心操作（生成周报、查看热度）不超过 2 次点击 |
| **专业感 + 克制** | 深色侧边栏 + 白色内容区，去除花哨装饰，对标 Bloomberg/Wind |
| **渐进式披露** | 概览 → 详情 → 操作，不一次性暴露所有功能 |

## 2. 全局布局重构

### 2.1 当前布局（问题分析）

```
┌─────────────────────────────────────────────┐
│           Header (品牌 + 标语)               │  ← 占用垂直空间，无功能价值
├─────────────────────────────────────────────┤
│  资讯浏览 | 任务管理 | 股票分析 | 研报分析    │  ← 顶部导航，平铺所有功能
├─────────────────────────────────────────────┤
│                                             │
│              内容区域                         │
│                                             │
└─────────────────────────────────────────────┘
```

**问题：**
- Header 占 80px 高度，纯装饰
- 顶部导航无层级，新增功能后会拥挤
- 无全局状态展示（今日采集量、热度异动等）
- 无法同时看到多个模块的概览

### 2.2 新布局

```
┌──────┬──────────────────────────────────────────────┐
```

## 6. 交互状态与反馈

| 场景 | 反馈方式 |
|------|----------|
| 页面加载 | 骨架屏（Skeleton），不用 loading spinner |
| 操作成功 | 顶部 Toast，绿色，2s 自动消失 |
| 操作失败 | 顶部 Toast，红色，需手动关闭 |
| 数据为空 | 空状态插图 + 引导文案 |
| 周报生成中 | 进度条 + 预计时间 |
| 热度异动 | 右上角通知铃铛红点 + 下拉面板 |
