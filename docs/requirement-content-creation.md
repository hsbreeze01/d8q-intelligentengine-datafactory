# REQ-001: 资讯内容创作与发布系统

## 1. 需求概述

基于已采集的资讯数据，支持不同风格的内容加工，输出到指定文章目录，并支持自动发布到社交平台。

## 2. 功能需求

### 2.1 内容创作引擎
- 读取指定时间范围的资讯数据（24h/7d/30d）
- 按主题聚合资讯
- 支持多种文章风格：资讯速递、深度分析、投资周报
- 输出 Markdown 格式文章

### 2.2 文章目录结构
```
articles/
  └── {主题}/
      └── {YYYY-MM-DD}/
          └── {YYYY-MM-DD}_{title}.md
```

### 2.3 任务管理扩展
在现有任务管理 Tab 中新增两种任务类型：

#### 创作任务
- 选择：主题、文章风格
- 频率：每日/每周/每月
- 自动拉取对应时间范围的资讯作为创作材料
- 生成文章到 articles 目录

#### 发布任务
- 选择：主题、发布渠道（小红书）
- 频率：每日/每周/每月
- 发布账号：内置配置
- 从 articles 目录读取最新文章发布

## 3. 技术设计

### 3.1 后端 API
- POST /api/content/create - 手动触发创作
- GET  /api/content/articles - 查看已生成文章列表
- POST /api/tasks (type=creation) - 创建创作任务
- POST /api/tasks (type=publish) - 创建发布任务

### 3.2 文章风格模板
| 风格 | 说明 | 适用频率 |
|------|------|----------|
| news_brief | 资讯速递，简洁列表 | 每日 |
| deep_analysis | 深度分析，带观点 | 每周 |
| investment_weekly | 投资周报，数据+趋势 | 每周/每月 |

### 3.3 数据流
```
资讯DB -> 按主题+时间筛选 -> 风格模板渲染 -> Markdown文件 -> 发布平台
```
